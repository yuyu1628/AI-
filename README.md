# 多Agent长篇小说工作流 (Multi-Agent Novel Pipeline)

本项目是一个高度工程化的 AI 长篇小说创作框架，旨在通过**多智能体（Multi-Agent）协作**与**状态机管理**，实现网文小说的工业级流水线生产。

区别于传统的单次 Prompt 续写，本项目模拟了真实的人类创作团队和编辑部流转过程。通过全局状态机与动态记忆提取，彻底解决大模型长文本生成的“记忆遗忘”、“设定崩塌”以及“AI味浓重”等痛点。

## ✨ 核心技术特性

* **🧠 全局状态机 (World Bible)：** 动态维护 `world_bible.json`，实时更新人物档案（生死/境界/性格）、核心道具归属与世界观法则。
* **🗂️ 动态滑动记忆 (Rolling Memory)：** 采用 `rolling_memory.json` 维持最近 5 章的极简摘要，并提取上一章末尾 300 字作为情绪无缝衔接的锚点。
* **🕵️‍♂️ 战前情报提取 (Context RAG)：** 动笔前精准抽取设定集中与本章强相关的出场人物与法则，拒绝无效“报菜名”和幻觉设定。
* **⚖️ 双盲审判庭 (Double-blind Review)：** * **排版编辑**：严查行文呼吸感与长短句节奏，剔除高频网文套路词及 AI 翻译腔。
  * **逻辑编辑**：校验设定一致性，确保主角降维打击的压迫感与反派破防的层次感。
  * **定向精修**：针对打回意见，调用专职修改模型进行局部替换与润色，而非盲目重写全文。
* **🚦 人类熔断机制 (Human-in-the-loop)：** 关键节点触发等待，允许人类作者手动介入修改 JSON 设定后放行。

## ⚙️ 七大核心模块 (Workflow)

1. **全书大纲**：推演 10 卷百万字长线大纲。
2. **深度设定扩充**：补全人物、道具、势力等初始世界观。
3. **卷大纲拆解**：拆解单卷 20 章详细动作线、情绪流与结尾卡点。
4. **战前情报 RAG**：按需提取设定集切片。
5. **核心写手创作**：融合前情提要、动态约束输出正文初稿。
6. **双盲审核与精修**：排版审与逻辑审双重把关，不达标则打回精修。
7. **状态机流转**：提取本章新增事实，烙印至全局档案库。

## 🚀 安装与运行

**1. 安装依赖**
请确保你的 Python 环境中安装了以下核心库：
> pip install openai rich python-dotenv

**2. 配置环境变量**
在项目根目录创建一个 `.env` 文件，填入你的 API 配置（具体格式可参考 `.env.example` 文件）：
> API_KEY_BASE=你的真实API_KEY
> BASE_URL_BASE=你的API基础地址

**3. 启动流水线**
> python main.py

跟随终端提示输入你的核心脑洞，并在提示 `>>> 🚦 熔断确认点 🚦 <<<` 时，确认无误后按 Y 继续。

## 📂 核心文件说明

* `main.py`：主程序执行入口。
* `world_bible.json`：全局状态机文件（程序运行时自动生成并维护）。
* `rolling_memory.json`：滑动窗口记忆缓存（程序运行时自动生成并维护）。
* `.env.example`：环境变量配置参考模板。
